🔐 Autenticación y Sesión

Flujo de login
  • Usuario envía credenciales al backend
  • Backend valida y devuelve JWT firmado
  • Next.js guarda el JWT en cookie httpOnly (no accesible desde JS)
  • Cookie mecaflow_tenant guarda el nombre del taller (readable)
  • Proxy en /api/proxy/[...path] inyecta el JWT en cada request
Claims del JWT
  • tenantId — ID del taller del usuario
  • username — nombre de usuario
  • role — Admin, User o SuperAdmin
  • Issuer y Audience: mecaflow
  • Cada query del backend filtra por tenantId automáticamente

🏢 Arquitectura Multi-Tenant

Cada taller (tenant) es completamente aislado. Los datos nunca se mezclan entre talleres. El tenantId se extrae del JWT en cada request y filtra todas las queries automáticamente.

📡 Servicio de Health Check WhatsApp

WhatsAppHealthService — BackgroundService
  • Se inicia 30 segundos después del boot de la app (para no interferir con el arranque)
  • Verifica el estado de la instancia de WhatsApp cada 5 minutos
  • Si el estado es open → no hace nada, resetea el contador de fallos
  • Si el estado es close → llama a /instance/connect/{instance} para reconectar
  • Espera hasta 15 segundos (3 intentos × 5s) verificando si alcanzó estado open
  • Si reconecta solo → logea éxito. Si no → logea advertencia con contador de intentos fallidos
  • Nunca lanza excepciones (todo el bloque está en try/catch) para no derribar la app
